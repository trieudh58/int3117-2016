# Nhận xét bài tập của Nguyễn Văn Tuynh

## Nhận xét BT1
- Được:
```
  + `README.md` có mô tả về hàm cần kiểm thử và hàm kiểm thử.
  + Mã nguồn khá ổn, trừ một số chỗ nhỏ spacing và style chưa ổn ( Ví dụ : if(s.length()<6&&s.length()>0)  hơi khó nhìn, nên thêm space).
  + Số lượng ca kiểm thử đủ và ổn. 
 ``` 
- Chưa được:
```
  + Nên để file code và file test vào thư mục riêng . 
  + Đặt tên file chưa chuẩn( Practive.java  thì file test phải là PractiveTest.java ).
  + Readme hơi đơn giản , không có ảnh của test bao phủ, đồ thị chương trình.
  + Giới thiệu bài toán, phương án giải quyết hơi ngắn gọn ,chưa rõ ràng.
```
## Nhận xét BT2
- Được:
```
  + Có mô tả bài toán.
  + Số lượng ca kiểm thử khá đầy đủ.  
  + Code khá dễ đọc.
  + Có sinh các ca kiểm thử.
 ```
- Chưa được:
```
  + Nên để file code và file test vào thư mục riêng .
  + Đặt tên file chưa chuẩn ( Password.java  thì file test phải là PasswordTest.java ). 
  + Readme hơi đơn giản nhưng nên có ảnh để minh họa , trình bày nên có các heading , code nên để trong space riêng để dễ nhìn hơn .
  + Chưa mô tả cách áp dụng MCDC và xây dựng các ca kiểm thử dựa trên phương pháp MCDC ( ở đây bạn nói áp dụng luôn MCDC để sinh ra các ca kiểm thử).
  + Không có kết quả kiểm thử, test bao phủ.  
```
## Nhận xét BT3
- Được:
```
  + Nêu ra cách giải quyết bài toàn rõ ràng ,đầy đủ
  + Có nêu ra các hàm kiểm thử .
  + Code rõ ràng, kiểm thử đầy đủ
```
- Chưa được:
```
  + Chưa mô tả bài toán ( có lẽ đã nêu ở BT2 nhưng vẫn nên update lại để readme rõ ràng hơn).
  + Chưa cập nhật mã nguồn - test BT3 ( mã nguồn thì có thể vẫn giống bài 2 nhưng nên update lên).
  + Chưa so sánh với MCDC .
  + Không có kết quả của test bao phủ, đồ thị DU-Path.
```
